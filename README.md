# BSA12_NFRs

Проект по выявлению, классификации и детализации нефункциональных требований для двух кейсов: онлайн-запись в барбершоп (BRB) и сервис доставки (DLV).

## 🛠 Навыки
- Non‑Functional Requirements (NFR) elicitation
- Performance, Availability, Reliability, Scalability, Localization, Security
- Logging & Password policies
- Multi‑factor authentication

## 📋 Описание проекта
В ходе работы я выделила нефункциональные требования для двух сервисов, классифицировала их по типам, определила приоритеты, а также детализировала атрибуты производительности, доступности, надёжности, масштабируемости, локализации и безопасности.

### Что именно было сделано:
1. **Выделение НФТ из текста (ex00)**: Определила номера предложений, содержащих нефункциональные требования (2,4,5,7,8,9).
2. **Классификация НФТ (ex01)**: Для 11 требований указала тип (производительность, безопасность, долговечность, доступность, локализация, масштабируемость, удобство использования, совместимость).
3. **Приоритеты НФТ (ex02)**: Для задач 1 и 2 определила приоритет (1–5) для 7 типов НФТ.
4. **Атрибуты производительности (ex03)**: Описала ключевые сценарии, количество пользователей, пиковые периоды и время отклика для операций BRB и DLV.
5. **Атрибуты доступности и надёжности (ex04)**: Для BRB и DLV описала SLA, время работы, RTO, RPO, целостность данных.
6. **Атрибуты масштабирования и локализации (ex05)**: Для BRB и DLV описала потребность в масштабировании и локализации.
7. **Атрибуты безопасности (ex06)**: Для BRB и DLV описала требования к паролям, MFA и логированию.
